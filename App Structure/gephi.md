# 📊 Gephi – Complete Notes (Android Reverse Engineering & Call Graph Analysis)

---

# What is Gephi?

**Gephi** is a free, open-source **graph visualization and network analysis** tool used to explore relationships between objects. It is widely used in:

* Android Reverse Engineering
* Malware Analysis
* Social Network Analysis
* Software Engineering
* Data Science
* Cybersecurity Research
* Network Traffic Analysis

Unlike reverse engineering tools (JADX, Androguard, Ghidra), Gephi **does not analyze APKs directly**. Instead, it visualizes graph data (e.g., call graphs) generated by other tools.

---

# Why Use Gephi in Android Reverse Engineering?

Large Android applications often contain:

* Thousands of classes
* Tens of thousands of methods
* Hundreds of thousands of method calls

Reading this manually is difficult.

Gephi transforms this into an **interactive graph**, allowing you to:

* Visualize method relationships
* Identify central methods
* Detect application modules
* Discover attack surfaces
* Understand application architecture

# Graph Terminology

## Node

Represents an object.

In Android:

* Method
* Class
* Activity
* Service

Example:

```
LoginActivity.login()
```

---

## Edge

Represents a relationship.

In Android:

```
Method A
      │
      ▼
Method B
```

means

```
A calls B
```

---

## Directed Graph

Android call graphs are directed.

```
A → B
```

does not mean

```
B → A
```

---

# Example

```
MainActivity.onCreate()
           │
           ▼
LoginActivity.login()
           │
           ▼
DatabaseHelper.checkPassword()
           │
           ▼
SQLiteDatabase.query()
```

Each box is a node.

Each arrow is an edge.

# Layout Algorithms

## 1. ForceAtlas2 ⭐⭐⭐⭐⭐

Most popular.

Good for:

* Medium graphs
* Community detection
* Software architecture

---

## 2. Yifan Hu ⭐⭐⭐⭐⭐

Best for large graphs.

Handles:

* 10,000+
* 100,000+
* Million-node graphs

---

## 3. Fruchterman-Reingold

Classic force-directed layout.

Suitable for medium-sized graphs.

---

## 4. OpenOrd

Designed for extremely large graphs.

Excellent for malware analysis and APKs with many methods.

# Graph Statistics

## Degree

Number of connections.

High-degree methods often include:

```
MainActivity.onCreate()
Application.onCreate()
Utility methods
```

---

## In-Degree

Number of incoming calls.

Useful for identifying:

* Frequently used methods
* Shared utility functions

---

## Out-Degree

Number of outgoing calls.

Often indicates:

* Controller methods
* Dispatchers
* Entry points

---

## Betweenness Centrality ⭐⭐⭐⭐⭐

Measures how often a node lies on the shortest paths between other nodes.

High betweenness often indicates:

* Authentication logic
* Database managers
* Network layers
* Security checks

---

## Closeness Centrality

Measures how quickly a node can reach others.

Useful for identifying core components.

---

## Eigenvector Centrality

Measures influence based on connections to other important nodes.

Useful for finding highly influential methods.

---

## Modularity ⭐⭐⭐⭐⭐

Groups related nodes into communities.

Typical Android clusters:

```
UI

Networking

Database

Crypto

Game Engine

Audio

Rendering
```

# Common File Formats

| Format     | Description                           |
| ---------- | ------------------------------------- |
| `.gexf`    | Native Gephi XML format (recommended) |
| `.graphml` | XML-based graph format                |
| `.gml`     | Graph Modeling Language               |
| `.csv`     | Node/edge tables                      |
| `.net`     | Pajek network format                  |

For Android call graphs, **`.gexf`** is generally the preferred format.

# Common Workflow with Androguard

```python
from androguard.misc import AnalyzeAPK
import networkx as nx

a, d, dx = AnalyzeAPK("app.apk")

cg = dx.get_call_graph()

nx.write_gexf(cg, "callgraph.gexf")
```

Open `callgraph.gexf` in Gephi, compute statistics, filter library packages, apply a layout, and investigate high-centrality methods in JADX or Smali.

---

# Best Practices

* Filter out framework and third-party library packages before analysis.
* Use **Yifan Hu** or **OpenOrd** for very large graphs, then refine with **ForceAtlas2**.
* Size nodes by **Degree** or **Betweenness Centrality** to highlight important methods.
* Color nodes by **Modularity** to reveal functional clusters.
* Cross-reference interesting nodes with JADX, JD-GUI, or Smali to inspect the underlying code.
* Combine Gephi with Androguard's string, class, method, and cross-reference analysis for a more complete understanding of the application.
