# JADX (Java Decompiler for Android)

**JADX** is a free and open-source Android decompiler that converts **Dalvik Executable (DEX) bytecode** into **human-readable Java source code**. It is one of the most widely used tools for **Android reverse engineering, application analysis, malware research, debugging, and security testing**.

Android applications are distributed as **APK (Android Package)** files, which contain compiled **DEX bytecode**. Since DEX is not designed to be read by humans, JADX reconstructs it into readable Java code, making it easier to understand an application's functionality.

---

# Why Use JADX?

JADX helps security researchers, developers, and reverse engineers to:

- Analyze the internal logic of Android applications.
- Understand the behavior of closed-source apps.
- Perform Android security assessments.
- Investigate malware and suspicious applications.
- Discover hardcoded secrets such as API keys or credentials.
- Study authentication, encryption, and networking implementations.
- Debug and inspect applications during development.

---

# Supported Input Formats

JADX supports several Android and Java-related file formats.

| File Type | Description |
|-----------|-------------|
| `.apk` | Android application package |
| `.dex` | Dalvik Executable bytecode |
| `.aab` | Android App Bundle |
| `.aar` | Android library package |
| `.jar` | Java archive |
| `.class` | Compiled Java bytecode |
| `.zip` | Archive containing supported files |

---

# Output Generated by JADX

After decompilation, JADX produces:

| Output | Description |
|---------|-------------|
| Java source code | Decompiled, readable Java classes |
| AndroidManifest.xml | Application configuration |
| Resources | XML layouts, images, strings, assets |
| Package hierarchy | Organized project structure |
| Smali code (optional) | Low-level Android bytecode representation |

---

# Components of JADX

JADX consists of two primary components.

| Component | Purpose |
|-----------|---------|
| **jadx** | Command-line decompiler for automation and batch processing |
| **jadx-gui** | Graphical interface for interactive analysis |

---

# Difference Between `jadx` and `jadx-gui`

| Feature | `jadx` | `jadx-gui` |
|---------|--------|------------|
| Interface | Command-line | Graphical User Interface (GUI) |
| Best for | Automation and scripting | Interactive code exploration |
| Navigation | No | Yes |
| Code Search | Limited | Powerful global search |
| Batch Processing | Yes | No |
| Reverse Engineering | Yes | Yes |

---

# Key Features

JADX provides several features that simplify Android application analysis.

- Decompiles DEX bytecode into readable Java code.
- Decodes Android resources and manifests.
- Displays complete package and class hierarchy.
- Built-in project-wide search.
- Cross-reference navigation.
- View Java and Smali code side-by-side.
- Built-in deobfuscation to improve readability.
- Export decompiled projects.
- Syntax highlighting and code navigation.

# Basic Usage

## Decompile an APK

```bash
jadx app.apk
```

This creates a directory containing the decompiled Java source code and extracted resources.

Specify a custom output directory:

```bash
jadx -d output app.apk
```

---

## Launch the Graphical Interface

```bash
jadx-gui
```

Or directly open an APK:

```bash
jadx-gui app.apk
```

Inside the GUI you can:

1. Browse packages and classes.
2. View decompiled Java source.
3. Search classes, methods, and strings.
4. Jump to definitions and references.
5. Compare Java and Smali code.

# Typical Reverse Engineering Workflow

A common Android reverse engineering workflow using JADX is:

### Step 1: Obtain the APK

```bash
adb pull /data/app/com.example.app/base.apk
```

### Step 2: Open the APK

```bash
jadx-gui base.apk
```

### Step 3: Explore the Application

Analyze important Android components such as:

- Activities
- Services
- Broadcast Receivers
- Content Providers
- Fragments
- Utility classes

### Step 4: Review Security-Relevant Code

Inspect:

- Login implementation
- API communication
- Data storage
- Encryption
- Authentication
- Network requests

---

# Limitations

Although JADX is powerful, it has some limitations.

| Limitation | Description |
|------------|-------------|
| Obfuscated applications | Decompiled code may still be difficult to understand |
| Incomplete reconstruction | Decompiled Java may not exactly match the original source |
| Native libraries | Cannot decompile `.so` native binaries |
| Heavy obfuscation | Some methods may not decompile correctly |

# Summary

JADX is one of the most popular Android reverse engineering tools for converting APK and DEX files into readable Java source code. It provides both a **command-line interface (`jadx`)** for automation and a **graphical interface (`jadx-gui`)** for interactive analysis. With features such as Java decompilation, resource decoding, project-wide search, code navigation, and deobfuscation, JADX is widely used in Android application analysis, malware research, debugging, and security assessments.
