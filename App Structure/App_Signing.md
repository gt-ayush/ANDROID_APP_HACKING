# 🔐 1. App Signing in Android

## 🔹 Definition

**App Signing** is the process of digitally signing an APK using a cryptographic key before installation.

---

## 🎯 Purpose

* **Authenticity** → verifies developer identity
* **Integrity** → prevents tampering
* **Secure Updates** → same key required

---

## 🔐 Importance

| Purpose   | Description                         |
| --------- | ----------------------------------- |
| Identity  | Identifies developer                |
| Integrity | Prevents unauthorized modification  |
| Updates   | Same key required for updates       |
| Security  | Enables signature-based permissions |

---

## 🔑 Types of Keys

### 1️⃣ Debug Key

* Auto-generated
* Used for development
* Not secure ❌

---

### 2️⃣ Release Key

* Created by developer
* Used in production
* Must be secure ✅

---

## 🔄 Signing Process

```text
Code → Build APK → Sign APK → Install
```

---

## 🔍 Key Concepts

* **Keystore (.jks)** → stores private keys
* **Alias** → identifier for key
* **Certificate** → public key
* **Private Key** → used for signing

---

## ⚠️ Rules

* Same key required for updates
* Unsigned APK → cannot install
* Same signature → apps can share data

---

## 🔐 Security Insight

* Repacked APK must be re-signed
* Signature mismatch → install fails
* Used in reverse engineering

---

# 🔑 2. keytool (Key Generation)

## 🔹 Definition

`keytool` is used to **generate and manage keystores and keys**

---

## ⚙️ Generate Keystore (Correct Command)

```bash
keytool -genkeypair -v \
-keystore ~/android-app-hack.keystore \
-alias myalias \
-keyalg RSA \
-keysize 2048 \
-validity 365
```

---

## 🔍 Important Points

* Alias must be **single word** ❗
* Password must be remembered
* Keystore must be सुरक्षित 🔐

---

## 📌 Other Commands

### List keys

```bash
keytool -list -v -keystore mykeystore.jks
```

### Delete key

```bash
keytool -delete -alias myalias -keystore mykeystore.jks
```

# 🔧 3. APK Signing Tools

---

## 🔹 1️⃣ jarsigner

### 📌 Description

* Java tool for signing APK
* Uses **v1 signature**

### ⚙️ Command

```bash
jarsigner -keystore mykeystore.jks app.apk myalias
```

### ❗ Status

* Deprecated ❌

---

## 🔹 2️⃣ zipalign

### 📌 Description

* Optimizes APK for performance

### ⚙️ Command

```bash
zipalign -v 4 input.apk aligned.apk
```

---

## 🔹 3️⃣ apksigner (Recommended)

### 📌 Description

* Official Android signing tool

### ⚙️ Command

```bash
apksigner sign --ks mykeystore.jks aligned.apk
```

---

## ✔ Features

* Supports v1, v2, v3
* Secure and modern

---

# 🔄 Complete Workflow (VERY IMPORTANT)

```bash
apktool b app → unsigned.apk
        ↓
zipalign → aligned.apk
        ↓
apksigner → signed.apk
```

---

## ⚠️ Common Mistakes

* ❌ Wrong alias
* ❌ Signing before zipalign
* ❌ Modifying signed APK
* ❌ Losing keystore

---

# 📊 Tool Comparison

**Android app signing uses cryptographic keys (generated by keytool) and tools like zipalign and apksigner to ensure secure, authentic, and installable APKs.**

### 🔐 `-sigalg` (Signature Algorithm) — Simple Note

## 🔹 Definition

`-sigalg` specifies the **algorithm used to create the digital signature**.

👉 It defines **how the data is signed using the private key**

---

## ⚙️ Example

```bash
keytool -genkeypair -sigalg SHA256withRSA ...
```

---

## 🧠 Meaning

```text
SHA256withRSA
```

=

* **SHA-256** → hashing algorithm
* **RSA** → encryption/signing algorithm

---

## 🔍 Common Values

| Value         | Meaning                |
| ------------- | ---------------------- |
| SHA1withRSA   | Old (weak ❌)           |
| SHA256withRSA | Modern & secure ✅      |
| SHA512withRSA | Stronger (less common) |

---

## ⚠️ Important Points

* If not specified → default is used
* SHA1 is deprecated ❌
* Always prefer **SHA256withRSA** ✅

---

## 🔐 Where It Is Used

* `keytool` (key generation)
* `jarsigner` (APK signing)

---

## 🧠 One-line (Exam Ready)

**`-sigalg` defines the cryptographic algorithm used to generate the digital signature of an APK or key.**
